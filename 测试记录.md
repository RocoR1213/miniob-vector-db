### 测试 SQL

```sql
CREATE TABLE v1(id int, embedding vector(3));
INSERT INTO v1 VALUES(1, STRING_TO_VECTOR('[1,2,3]'));
INSERT INTO v1 VALUES(2, STRING_TO_VECTOR('[4,5,6]'));
INSERT INTO v1 VALUES(3, STRING_TO_VECTOR('[1.5,-2,3.25]'));
INSERT INTO v1 VALUES(4, STRING_TO_VECTOR('[7,8,9]'));
INSERT INTO v1 VALUES(5, STRING_TO_VECTOR('[2,2,2]'));

CREATE VECTOR INDEX idx1 ON v1(embedding) WITH (lists=2, probes=1);
CREATE VECTOR INDEX idx_bad ON v1(embedding) WITH (lists=0, probes=2);

SELECT * FROM v1;
SELECT VECTOR_TO_STRING(embedding) FROM v1;
SELECT id, DISTANCE(embedding, STRING_TO_VECTOR('[0,0,0]'), 'EUCLIDEAN') FROM v1;
```

### 通过情况

| 测试项 | 结果 | 说明 |
|--------|------|------|
| CREATE TABLE + 5×INSERT | SUCCESS | 建表插入正常 |
| `CREATE VECTOR INDEX ... (lists=2, probes=1)` | **SUCCESS** | K-Means 训练成功，索引文件落盘 |
| `CREATE VECTOR INDEX ... (lists=0, probes=2)` | FAILURE | lists≤0 被拦截 |
| SELECT * / VECTOR_TO_STRING / DISTANCE | 正确 | 数据完整性验证通过 |

### 已知问题

- 退出时 `Failed to flush index's pages`：`IvfflatIndex::sync()` 返回 `UNIMPLEMENTED`，索引数据已在 `create()` 中写入，重启后通过 `open()` 加载。不影响功能。
- `delete_entry` / `sync` 暂返回 `UNIMPLEMENTED`，后续按需实现。
